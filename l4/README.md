# Обработка изображений: PNG size to JSON

Данный проект демонстрирует реализацию одного и того же веб-функционала на двух языках программирования — **Go** и **Python**.

## Общая задача

Реализовать веб-приложение, которое:

- По маршруту `/login` возвращает JSON с логином: `{"author": "1155281"}`.
- По маршруту `/size2json` принимает PNG-изображение через `multipart/form-data` и возвращает JSON с его размерами: `{"width":123,"height":456}`.
- При передаче не-PNG файла возвращает: `{"result":"invalid filetype"}`.
- Все ответы сопровождаются заголовком `Content-Type: application/json`.

## Реализация на Go (`main.go`)

### Технологии
- Стандартная библиотека Go: `net/http`, `image/png`.
- Никаких внешних зависимостей.

### Особенности
- Высокая производительность и низкое потребление памяти.
- Компилируется в один бинарный файл — легко разворачивать.
- Явная обработка MIME-типов и парсинга multipart-форм.
- Обработка изображений только через `png.Decode`, что делает проверку типа строгой.

### Запуск
```bash
go run main.go
```
Сервер стартует на `http://localhost:8080`.

## Реализация на Python (`app.py`)

### Технологии
- `FastAPI` — современный асинхронный веб-фреймворк.
- `Pillow` — библиотека для обработки изображений.

### Особенности
- Более высокий уровень абстракции: `UploadFile`, автоматическая валидация.
- `Pillow` поддерживает множество форматов, но при ошибках декодирования выбрасывает `UnidentifiedImageError`, что используется для фильтрации не-PNG.
- Быстрое прототипирование, богатая экосистема.
- Требует установки зависимостей: `pip install fastapi pillow uvicorn`.

### Запуск
```bash
uvicorn app:app --reload --port 8080
```
Сервер стартует на `http://localhost:8080`.

## Сравнение Go и Python

| Критерий | Go | Python |
|--------|-----|--------|
| Язык | Компилируемый | Интерпретируемый |
| Производительность | Высокая | Ниже (особенно при нагрузке) |
| Зависимости | Стандартная библиотека | Требует установки внешних пакетов |
| Простота развертывания | Одно бинарное приложение | Нужен интерпретатор и `requirements.txt` |
| Обработка ошибок | Явная проверка `err != nil` | Через исключения (`try-except`) |
| Типизация | Статическая | Динамическая |
| Веб-фреймворк | Нет (встроенный `net/http`) | FastAPI (высокоуровневый) |
| Обработка изображений | Только PNG через `image/png` | Любые форматы через `Pillow` |
